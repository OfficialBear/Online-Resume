# Axiom

**A product-grade portfolio for backend engineers.**

Designed to communicate system-level thinking, engineering ownership, and measurable impact — within seconds.

---

## Built for clarity

Most portfolios list technologies.
This one communicates **how systems are designed, built, and improved**.

* From implementation → to architecture
* From features → to impact
* From code → to decisions

---

## Why it exists

Hiring decisions are fast.

This project is built around a single idea:

> Engineering ability should be understood quickly, without explanation.

---

## What it shows

* **System Design Thinking**
  Clear structure of how systems are organized and scaled

* **Ownership & Responsibility**
  Focus on what was actually built and owned

* **Performance & Optimization**
  Real improvements with measurable results

* **Engineering Trade-offs**
  Not just what was done, but why

---

## Experience, not listing

Instead of:

```
Used Redis to improve performance
```

This project presents:

```
Introduced Redis caching for hotspot data,
reducing database load by 40% and improving API response time from 320ms to 120ms
```
